## What Was Done This Session

### 1. **Speech vs Display Text Separation** (jarvis/brain/speech_text.py)
- Added `speech_text` field to `BrainResponse` (jarvis/brain/base.py)
- Created shared utility `generate_speech_text()` that:
  - Strips markdown/code blocks
  - Announces tool calls as "Używam narzędzia X" instead of reading full text
  - Truncates to ~200 chars for natural TTS
- Updated adapters: `MockBrainAdapter`, `ClaudeCliAdapter` (blocking + streaming), `CodexCliAdapter`

### 2. **PTT Double-Click Bug Fix** (jarvis/panel/hotkey.py)
- **Problem**: Rapid double-click on PTT (left_cmd+left_shift) created duplicate leases, mic stayed on
- **Fix**: 
  - `HotkeyEdgeDetector`: Added 150ms debounce on "down" events only (no debounce on "up")
  - `PttHotkeyClient`: Added client-side `threading.Lock` - ignores new requests while HTTP in-flight
  - Tests updated/verified in `tests/test_panel_hotkey.py`

### 3. **Trusted Scopes + Voice Auto-Approve** (jarvis/config.py, jarvis/tools/permissions.py)
- `TrustedScope` config: `[security.trusted_scopes]` array with `name`, `path`, `tools`, `ttl_minutes`
- `voice_auto_approve_tools` boolean: VOICE_COMMAND gets ALLOW for mutation tools within approved_roots
- `ToolPermissionPolicy` now allows MODEL_ORIGINATED and VOICE_COMMAND to get ALLOW (not APPROVAL_REQUIRED) when in trusted scope
- Passes through daemon: `create_daemon_app` → `ToolPermissionPolicy(trusted_scopes=..., voice_auto_approve=...)`

### 4. **One-Click Approve + Execute** (jarvis/panel/assets/app.js)
- Non-memory tools: button now "Zatwierdź i wykonaj" → calls both `/approve` + `/execute` sequentially
- Eliminates separate approve→wait→execute clicks

### 5. **Approval Continuation Fix** (jarvis/daemon/app.py, jarvis/turns/orchestrator.py)
- `_execute_approved_tool_locked` calls `continue_after_tool_result` which injects `tool_result` into next turn
- Brain receives tool result context, doesn't re-request same tool

## Config Example for Trusted Scopes

```toml
[security]
approved_roots = ["/Users/ozzy/projects"]
trusted_scopes = [
  { name = "jarvis-dev", path = "/Users/ozzy/projects/jarvis", tools = ["file_write", "file_read", "shell_read"], ttl_minutes = 60 }
]
voice_auto_approve_tools = true
```

This gives MODEL_ORIGINATED + VOICE_COMMAND auto-approval for file_write/read/shell_read within `/Users/ozzy/projects/jarvis`.
